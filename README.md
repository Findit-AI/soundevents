## Overview

`soundevents` is a Cargo workspace that provides typed, ergonomic Rust building blocks for sound event recognition and audio taxonomy work. The long-term goal is a small, composable set of crates covering everything from raw ontology lookup to higher-level classification helpers.

## Crates

| Crate | Status | Description |
| --- | --- | --- |
| [`soundevents-dataset`](./soundevents-dataset) | **available** | Zero-allocation, `no_std`-friendly access to the full AudioSet ontology — 600+ entries with names, descriptions, hierarchy, aliases, and restrictions, baked in at compile time. See its [README](./soundevents-dataset/README.md) for usage. |
| [`soundevents`](./soundevents) | *planned* | Umbrella crate for the higher-level API. Not yet implemented. |

If you only need ontology lookup today, depend on `soundevents-dataset` directly.

## Repository layout

```text
soundevents/                # umbrella crate (placeholder)
soundevents-dataset/        # AudioSet ontology, codegen'd into &'static data
  ontology.json             # upstream source of truth
  src/generated.rs          # checked-in, generated by xtask
xtask/                      # codegen binary, run via `cargo xtask`
```

## Development

Regenerate `soundevents-dataset/src/generated.rs` from `ontology.json`:

```sh
cargo xtask codegen
```

Run the test suite:

```sh
cargo test
```
